Begrepp och Github

Github flow

GitHub-flow är utformat för att fördela arbetsbelastningen. Det är ett arbetsflöde som kan göras genom CLI (kommandorad) och GUI-applikationer (GitHub Desktop). Med hjälp av GitHub flow är det lättare att arbeta på distans och det fokuserar främst på att skapa branches. Varje individ kan skapa en branch, arbeta med den och distribuera den individuellt på GitHub.

Följande är Github-flödesfunktioner

  • Skapa en ny filial
  • Gör ändringar och lägg till Commits
  • Öppna en Pull Request
  • Review
  • Distribuera
  • Merge

Skapa en ny filial

Att skapa en branch och arbeta med den är huvudkonceptet för Github. Det ger oss en miljö för att göra förändringar utan att påverka huvud branch. När den nya branch är klar kan den granskas, diskuteras och slås samman med huvud branch. Gör ändringar och lägg till Commits. När du arbetar med ett projekt kan mängden utfört arbete läggas till den skapade branch. Den håller koll på arbetet. Varje commit måste ha ett meddelande som förklarar ändringarna som gjorts i grenen. Den kan återställas vid behov.

Gör ändringar och lägg till Commits

När du arbetar med ett projekt kan mängden utfört arbete läggas till den skapade branch. Den håller koll på arbetet. Varje commit måste ha ett meddelande som förklarar ändringarna som gjorts i grenen. Den kan återställas vid behov.

Öppna en Pull Request

En pull-request meddelar andra gruppmedlemmar om ändringarna som gjorts för att granska. Den är utformad så att alla gruppmedlemmar kan arbeta tillsammans.

Review

När en återkoppling tas emot krävs att de ändringar som gjorts drivs med nya kommentarer.

Distribuera

Utplaceringen görs efter den sista granskningen av pull-request. När den slutliga testningen är gjord distribueras den sedan till masterbranch.

Merge

Efter testning kan den muras till mastergrenarna

Flexbox

Flexbox är en endimensionell layoutmodell. Den fördelar objekten i rader och kolumner. Den har många egenskaper såsom flex-direction som kan vara rad eller kolumner, align-items, display, flex-direction, flex-wrap, justify-content och align-content. Det gör webbplatsen responsiv och tar bort blanksteg. Nedan finns en exampel på hur kan man använda Flexbox.


                        

                        .flex-container {
                            display: flex;
                            flex-flow: row wrap;
                        }



DOM (Document Object Model)

Dokumentobjektmodellen är trädstrukturen för en webbsida. När en sida laddas är HTML DOM-modellen konstruerad som ett träd av objekt. Den innehåller alla element och attribut för HTML-sidan. Med javascript är det lättare att ge funktionalitet till en webbplats. Det kan också ändra stilen på CSS.



Javascript är ett objektorienterat skriptspråk. Den kan utföra en enorm mängd funktioner. Med DOM är det lättare att manipulera sammanhanget för HTML och CSS (webbplats). De mest kända metoderna för DOM som används är EventListner(), getElementById(), getElementByClassName, getAttribute(), setAttribute() .etc.

Fin Kod

En bra kod i datorspråk är en uppsättning variabler, funktioner, loops, arrayer, listor och klasser arrangerade tillsammans på ett sätt som är lätt att läsa av andra programmerare. En bra torsk följer standarduppsättningen av regler som görs av programmerare att följa under programmering.

  • Läsbar
  • Kan läsas av människor
  • Självförklarande
  • Folk kan läsa den
  • Lätt att förstå
  • Det tar mindre tid att förstå det
  • Du kan läsa den, förstå den utan dokumentation
  • Förståeligt, nya utvecklare borde kunna förstå det
  • Lättläst som om det vore någon text
  • Rakt fram

Ramverk

Ett ramverk är en struktur som programvaran är byggd på. Ett ramverk gör det enkelt för programmerare att koda utan att bygga upp hela strukturen i programvaran från grunden. Det finns olika ramverk på datorspråk. Till exempel är Node, Angular och Ajax kända ramverk i javascript. Django, Flask och Bottle är ramverk som används i python för webbutveckling. Ramar används mest för att spara tid, undvika dubbletter av koder och skriva ren kod. Fördelar av

  • Säkrare kod
  • Enklare testning och felsökning
  • Undviker duplicerad kod
  • Ren och lätt anpassningsbar kod
  • Kunna fokusera på att skriva kod specifik för projektet
  • Kan förlängas
